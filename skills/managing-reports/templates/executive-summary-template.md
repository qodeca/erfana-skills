# Executive Summary Template

Use the BLUF (Bottom Line Up Front) method. Maximum 2 pages.

---

## Template

```markdown
# Executive summary

## Overall assessment

[1-2 sentences stating the main conclusion. Answer the client's primary question immediately.]

## Key findings

[3-5 most critical discoveries, ordered by priority]

1. **[Finding title]**: [One sentence summary with quantified impact]
2. **[Finding title]**: [One sentence summary with quantified impact]
3. **[Finding title]**: [One sentence summary with quantified impact]
4. **[Finding title]**: [One sentence summary with quantified impact]
5. **[Finding title]**: [One sentence summary with quantified impact]

## Key recommendations

[3-5 priority actions, ordered by urgency]

1. **[Recommendation title]**: [Action + owner + timeline]
2. **[Recommendation title]**: [Action + owner + timeline]
3. **[Recommendation title]**: [Action + owner + timeline]
4. **[Recommendation title]**: [Action + owner + timeline]
5. **[Recommendation title]**: [Action + owner + timeline]

## Expected outcomes

[1-2 sentences on what success looks like if recommendations are implemented]

## Next steps

[Optional: Immediate actions required before full implementation]

- [Immediate action 1]
- [Immediate action 2]
```

## Example

```markdown
# Executive summary

## Overall assessment

Acme Corp's technology ecosystem requires integration work to
achieve operational efficiency. The current state, characterized by disconnected
systems and manual processes, creates a 12% data error rate and consumes
180+ staff-hours monthly in duplicate data entry.

## Key findings

1. **System fragmentation**: 8 core systems operate independently with no
   automated data exchange, requiring manual re-entry of member, financial,
   and HR data.

2. **GymSoft limitations**: The member management system lacks API
   documentation and vendor support, blocking integration efforts.

3. **Excel dependency**: 15+ critical business processes rely on Excel
   spreadsheets, creating version control and data integrity risks.

4. **Mobile experience gaps**: The member-facing app receives 2.3/5 ratings
   with 45% of complaints related to booking failures and performance issues.

5. **Understaffing in IT**: The 3-person IT team cannot address technical
   debt while supporting 13 facilities and 50,000+ members.

## Key recommendations

1. **Implement GymSoft-HR Suite integration**: IT Director by Q2 2026.
   Eliminates 160 hours/month manual entry (USD 24,000 savings).

2. **Replace Excel with integrated solutions**: Operations Director by Q3 2026.
   Reduces data errors by 80% and improves audit compliance.

3. **Establish data governance committee**: COO by Q1 2026. Creates
   accountability for data quality across departments.

4. **Upgrade mobile app infrastructure**: IT Director by Q4 2026. Target
   4.0+ app store rating and <2% booking failure rate.

5. **Expand IT team capacity**: HR Director by Q1 2026. Add 2 FTEs to
   support integration roadmap execution.

## Expected outcomes

Full implementation of these recommendations will reduce operational costs by
approximately USD 450,000 annually, improve member satisfaction scores by an
estimated 15-20%, and position Acme Corp for scalable growth.

## Next steps

- Schedule leadership alignment meeting (within 2 weeks)
- Initiate vendor discussions with GymSoft (within 30 days)
- Establish budget allocation for Q1 2026 initiatives
```

## Guidelines

### What to Include

| Element | Content |
|---------|---------|
| Overall assessment | Main conclusion + critical context |
| Key findings | Top 3-5 by impact, quantified |
| Key recommendations | Top 3-5 by priority, with owner + timeline |
| Expected outcomes | Benefits of implementation |
| Next steps | Optional: immediate actions |

### What to Exclude

| Element | Reason |
|---------|--------|
| Methodology details | Put in appendix |
| All findings | Summary only; details in body |
| Background context | Client already knows this |
| Technical details | Put in body sections |
| New information | Everything must be in body first |

### Length Guidelines

**Rule:** The executive summary is ≤10% of the document, maximum 2 pages. The
buckets below are illustrative only.

| Document length | Executive summary length |
|-----------------|-------------------------|
| <20 pages | ~1 page |
| 20-50 pages | 1-2 pages |
| 50+ pages | 2 pages (≤10% of document) |

### Validation Checklist

- [ ] Leads with overall conclusion (first 1-2 sentences)
- [ ] Reader can understand main message without reading full report
- [ ] Findings are numbered and quantified
- [ ] Recommendations have owner + timeline
- [ ] All items use sentence case headings
- [ ] No Title Case in headings
- [ ] Length within guidelines
