# Section Template

Use the Pyramid Principle: lead with conclusion, support with evidence.

---

## Template

```markdown
## [Section title in sentence case]

[Opening paragraph: State the section's main conclusion/key message in 1-2 sentences.
This is the "answer" the reader is looking for.]

### [Subsection 1 title]

[Supporting point 1 with evidence. Explain and justify the first aspect of your
main message. Include data, examples, or analysis.]

### [Subsection 2 title]

[Supporting point 2 with evidence. Continue building the case for your main
message with additional supporting information.]

### [Subsection 3 title]

[Supporting point 3 with evidence. Add any remaining supporting points needed
to fully substantiate your main message.]

### Summary

[Optional: Brief recap if section is long. Transition to next section.]

---
```

## Example: Analysis Section

```markdown
## Technology integration assessment

Acme Corp's technology ecosystem operates as disconnected silos, creating
operational inefficiency and data quality issues. The analysis
identified 8 core systems with no automated integration, resulting in 180+
staff-hours monthly spent on manual data transfer.

### Current system landscape

The organization relies on 8 core technology systems for daily operations:

| System | Function | Vendor | Integration status |
|--------|----------|--------|-------------------|
| GymSoft | Member management | GymSoft | None |
| HR Suite | HR and payroll | HR Suite | None |
| Payroll Platform | HR compliance | Payroll Platform | None |
| Microsoft 365 | Productivity | Microsoft | Partial |
| AccountingApp | Accounting | AccountingApp Inc. | None |
| Intercom | Customer support | Intercom | None |
| Instagram/TikTok | Marketing | Meta/ByteDance | None |
| Custom Excel | Operations | Internal | N/A |

Only Microsoft 365 has partial integration with other systems through basic
calendar and email connectivity.

### Integration gaps identified

Three critical integration gaps create the majority of operational friction:

1. **Member data synchronization**: GymSoft, HR Suite, and Payroll Platform contain
   overlapping member and employee data that must be manually synchronized.
   Analysis of 200 records found 12% discrepancy rate.

2. **Financial data flow**: AccountingApp does not receive automatic feeds from
   GymSoft membership transactions or HR Suite payroll, requiring monthly
   manual reconciliation (estimated 40 hours/month).

3. **Customer communication**: Intercom has no access to member history from
   GymSoft, limiting support staff's ability to provide personalized service.

### Impact assessment

| Gap | Hours/Month | Error Rate | Financial Impact |
|-----|-------------|------------|------------------|
| Member data sync | 120 | 12% | USD 18,000 |
| Financial data | 40 | 5% | USD 6,000 |
| Customer support | 20 | N/A | USD 3,000 |
| **Total** | **180** | - | **USD 27,000/month** |

### Summary

The disconnected technology ecosystem costs approximately USD 324,000 annually
in direct labor costs, not including the hidden costs of errors, delays, and
customer dissatisfaction. The following section details findings and
recommendations for addressing these integration gaps.

---
```

## Structure Guidelines

### Opening Paragraph

| ✅ Do | ❌ Don't |
|------|---------|
| State conclusion first | Build up to conclusion |
| Give the "so what" | Start with methodology |
| Be specific | Be vague |
| Quantify if possible | Use general statements |

### Subsections

| ✅ Do | ❌ Don't |
|------|---------|
| Use descriptive titles | Use "Background" |
| Support main message | Introduce tangents |
| Include evidence | Make unsupported claims |
| Keep parallel structure | Mix formats |

### Tables and Visuals

| ✅ Do | ❌ Don't |
|------|---------|
| Reference in text | Leave unexplained |
| Add meaningful captions | Use generic captions |
| Keep simple (<10 columns) | Over-complicate |
| Ensure sentence case headers | Use Title Case |

## Section Types

### Analysis Section

```
1. Key finding statement
2. Evidence and data
3. Impact assessment
4. Implications
```

### Findings Section

```
1. Summary of findings
2. Individual findings (use finding template)
3. Cross-cutting themes
4. Prioritization rationale
```

### Recommendations Section

```
1. Summary of recommendations
2. Individual recommendations (use recommendation template)
3. Implementation roadmap
4. Resource requirements
```

### Background Section (if needed)

```
1. Context summary (brief)
2. Scope definition
3. Methodology overview
4. Limitations/constraints
```

---

## Validation Checklist

- [ ] Section opens with conclusion/key message
- [ ] Subsections support main message
- [ ] Evidence provided for claims
- [ ] Tables/figures referenced in text
- [ ] Section title in sentence case
- [ ] All subsection titles in sentence case
- [ ] No Title Case violations
- [ ] Paragraphs ≤5 sentences
- [ ] Sentences ≤40 words
- [ ] Logical flow to next section
