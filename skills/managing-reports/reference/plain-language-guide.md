# Plain Language Guide

Plain language ensures reports are accessible to all readers, including non-experts.

---

## Core Principles

1. **Write for the reader, not yourself**
2. **State your main point first**
3. **Use everyday words**
4. **Keep sentences short**
5. **Use active voice**
6. **Organize logically**

## Sentence Structure

### Length Guidelines

| Metric | Target | Maximum |
|--------|--------|---------|
| Average sentence length | 20 words | - |
| Maximum sentence length | - | 40 words |
| Paragraph length | 3-5 sentences | 5 sentences |

### Simplification Techniques

**1. Split long sentences**

❌ "The implementation of the new system, which was originally scheduled for Q1 2026 but has been delayed due to vendor constraints and budget limitations, will now commence in Q3 2026."

✅ "The new system implementation was scheduled for Q1 2026. Vendor constraints and budget limitations have pushed this to Q3 2026."

**2. Remove unnecessary words**

❌ "It is important to note that the system has the capability to..."
✅ "The system can..."

❌ "In order to achieve the desired outcome..."
✅ "To achieve this..."

**3. Use active voice**

❌ "The report was reviewed by the committee."
✅ "The committee reviewed the report."

## Abbreviation Rules

### When to Abbreviate

| Rule | Example |
|------|---------|
| Define on first use | "GymSoft Management System (PGMS)" |
| Don't abbreviate if used 1-2 times | Write full term |
| Industry-standard abbreviations | API, IT, ERP need no definition for technical audiences |
| Create abbreviation list if 10+ | Include as appendix |

### Abbreviation Stacking

❌ "The ERP-CRM-HCM integration requires API-level SSO authentication."
✅ "Integrating the enterprise resource planning, customer relationship, and human capital systems requires single sign-on authentication at the API level."

## Numbers and Dates

### Number Formatting

| Rule | Example |
|------|---------|
| Spell out 1-9 | "three systems" |
| Use numerals for 10+ | "15 facilities" |
| Use numerals with units | "8 hours", "5%" |
| Use thousands separator | "1,500 members" or "1 500 members" |
| Currency first | "USD 450,000" |

### Date Formatting

| Format | Example |
|--------|---------|
| Full date | "15 November 2025" |
| Month-year | "November 2025" |
| Quarter-year | "Q2 2026" |

❌ "recently", "soon", "last month"
✅ "October 2025", "by Q1 2026", "September 2025"

## Translation-Friendly Writing

For reports that may be translated:

1. **Use simple sentence structures** - Subject-verb-object
2. **Avoid idioms** - "hit the ground running" → "start quickly"
3. **Avoid cultural references** - May not translate
4. **Use consistent terminology** - Same word for same concept
5. **Avoid humor** - Often doesn't translate
6. **Use international date format** - "15 November 2025" not "11/15/2025"

## Readability Testing

### Flesch-Kincaid Grade Level

| Score | Interpretation |
|-------|----------------|
| 8-10 | Target range for business documents |
| 11-12 | Acceptable for technical audiences |
| 13+ | Too complex - simplify |

### Hemingway App Targets

- Grade level: 8-10
- Adverbs: Minimize
- Passive voice: <10%
- Hard to read sentences: <5%
- Very hard to read sentences: 0%
