# Quality Checklist Reference

Complete validation criteria for consulting reports before client delivery.

---

## Pre-Publish Checklist

### 1. Content Quality (30 points)

| # | Check | Points | Pass criteria |
|---|-------|--------|---------------|
| 1.1 | Findings have Five C's | 6 | ALL findings have Criteria, Condition, Cause, Consequence, Corrective Action |
| 1.2 | Claims supported by evidence | 6 | NO unsupported assertions |
| 1.3 | Recommendations are specific | 6 | ALL have action + owner + timeline |
| 1.4 | Recommendations address root cause | 6 | Each recommendation links to a cause |
| 1.5 | No unsupported opinions | 3 | NO "we believe" without evidence |
| 1.6 | Scope alignment | 3 | Content matches stated scope |

### 2. Style Quality (30 points)

| # | Check | Points | Pass criteria |
|---|-------|--------|---------------|
| 2.1 | Active voice | 6 | ≥90% of sentences active |
| 2.2 | No nominalizations | 6 | Zero from prohibited list |
| 2.3 | Sentence length average | 6 | ≤20 words average |
| 2.4 | Sentence length maximum | 6 | ZERO sentences >40 words |
| 2.5 | No redundant phrases | 3 | Zero from prohibited list |
| 2.6 | No jargon | 3 | Zero from forbidden list |

### 3. Capitalization (15 points) - CRITICAL

| # | Check | Points | Pass criteria |
|---|-------|--------|---------------|
| 3.1 | H1 headings | 3 | ALL sentence case (except cover title) |
| 3.2 | H2 headings | 3 | ALL sentence case |
| 3.3 | H3-H4 headings | 3 | ALL sentence case |
| 3.4 | List items | 3 | ALL sentence case |
| 3.5 | Table headers | 3 | ALL sentence case |

**ZERO TOLERANCE**: Any capitalization violation = FAIL entire check.

### 4. Formatting Quality (15 points)

| # | Check | Points | Pass criteria |
|---|-------|--------|---------------|
| 4.1 | Heading hierarchy | 3 | No skipped levels (H1→H3) |
| 4.2 | No stacked headings | 3 | Text between all headings |
| 4.3 | List item count | 3 | Lists have 2-7 items (fail: 1 item or 8+) |
| 4.4 | List parallel structure | 3 | All items same grammatical form |
| 4.5 | Tables referenced | 3 | All tables mentioned in text |

### 5. Precision Quality (10 points)

| # | Check | Points | Pass criteria |
|---|-------|--------|---------------|
| 5.1 | Dates specific | 2 | NO "recently", "soon" |
| 5.2 | Numbers specific | 2 | NO "several", "many" |
| 5.3 | Impacts quantified | 2 | ALL consequences have numbers |
| 5.4 | Abbreviations defined | 2 | ALL non-standard defined |
| 5.5 | Sources cited | 2 | All external references cited |

## Scoring and the delivery gate

### Delivery gate (all six validators must pass)

A report passes review only when **all six validators PASS**. Any violation in
any validator fails that validator and blocks delivery – there is no numeric
score that overrides a failure, and no validator is optional. Common blocking
violations:

| Condition | Validator |
|-----------|-----------|
| Any capitalization violation | capitalization |
| Any sentence >40 words | style |
| Missing owner or timeline on a recommendation | structure (Five C's) |
| Unsourced claim | precision |

### Quality score (advisory)

The 0-100 quality score is an **advisory** signal only – it does not gate
delivery. It is computed by the review-report agent using its weighted rubric
(capitalization and structure 20% each; style, formatting, precision, and
executive summary 15% each). Advisory rating bands:

| Score range | Rating |
|-------------|--------|
| 90-100 | Excellent |
| 80-89 | Good |
| 70-79 | Acceptable |
| 60-69 | Needs work |
| <60 | Poor |

## 5-minute pre-check

A rapid manual scan before the full six-validator review (this is a pre-check,
not a validation level):

- [ ] Executive summary is ≤10% of the document (max 2 pages)
- [ ] Each finding has a recommendation
- [ ] No obvious Title Case in headings
- [ ] No sentences wrap to 3+ lines
- [ ] All sections have content (no placeholders)

---

## Section-Specific Checks

### Executive Summary

- [ ] Leads with overall conclusion
- [ ] Contains 3-5 key findings (bullets)
- [ ] Contains 3-5 key recommendations (bullets)
- [ ] Length ≤10% of document (max 2-3 pages)
- [ ] Can be understood without reading full report
- [ ] No new information (all in body)

### Findings Section

- [ ] Each finding numbered for reference
- [ ] Each finding has severity rating
- [ ] Five C's complete for each
- [ ] Findings ordered by priority
- [ ] No duplicate findings
- [ ] Findings are MECE (no overlap, no gaps)

### Recommendations Section

- [ ] Each recommendation numbered
- [ ] Each has specific action verb
- [ ] Each has assigned owner
- [ ] Each has completion date
- [ ] Each has expected benefit
- [ ] Recommendations link to findings
- [ ] Recommendations are prioritized

### Appendices

- [ ] All referenced in main text
- [ ] Properly numbered (A, B, C)
- [ ] Contains supporting detail only
- [ ] No critical information hidden

## Review Workflow

### Step 1: Self-Review (Author)

1. Run spell check and grammar check
2. Check all sentences <40 words
3. Verify Five C's for each finding
4. Confirm all recommendations complete
5. Check heading capitalization

### Step 2: Peer Review

1. Validate evidence supports claims
2. Check logical flow
3. Verify MECE structure
4. Assess tone and diplomacy
5. Check for missing content

### Step 3: Quality Gate

1. Run full checklist (this document)
2. Confirm all six validators PASS (the delivery gate)
3. Record the advisory quality score
4. Document any accepted deviations

### Step 4: Final Approval

1. Confirm all six validators PASS (the score is advisory, not a gate)
2. Senior review sign-off
3. Client-ready confirmation
