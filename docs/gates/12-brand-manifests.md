# Gate 12 – brand manifests valid (schema-driven)

Validates every brand manifest under `skills/design-shared/brands/<id>/brand.json` against the `brand.schema.json` contract, plus the active-brand pointer and the production-brand allowlist. Convention-over-configuration: a folder is a brand iff it contains a `brand.json`. Folders prefixed with `_` are skipped as WIP.

The gate has two layers:

**Schema-driven shape validation.** Delegated to `scripts/_lib/json_schema_lite.py` – a stdlib-only minimal JSON Schema 2020-12 validator (covers the subset used by `brand.schema.json`: `type`, `required`, `properties`, `additionalProperties` (bool|schema), `items`, `pattern`, `minLength` / `maxLength`, `enum`, `const`, `oneOf`, `anyOf`, `minimum`, `maximum`). The schema is the single source of truth; typos like `"watermak"` instead of `"watermark"` are caught by `additionalProperties: false`. No `jsonschema` pip dependency.

**Cross-file invariants in the gate.** The schema cannot express these:

1. **`ACTIVE_BRAND` pointer file** exists at `skills/design-shared/brands/ACTIVE_BRAND`, contains a non-empty brand id, the id is on the `PRODUCTION_BRANDS` allowlist (currently `["erfana"]`), and that brand's folder exists. The `example-acme` placeholder is intentionally absent from the allowlist so it cannot become the active brand.
2. **`id` equals folder basename** for every manifest. Mismatch fails.
3. **Path traversal guard (manifest paths)** – every relative path in the manifest (`tokens`, `logos[].src`, `voice.guide`, `imagery.photoLibrary`, `imagery.illustrationStyle`, plus the v1.2 fields `imagery.backgroundLibrary`, `imagery.shapeLibrary`, `imagery.templateLibrary`, plus the v1.3 field `imagery.logoLibrary`) is resolved via `realpath`, then asserted to lie inside `realpath(repo)` before existence is checked. Absolute paths and `../` traversal that escape the repo are rejected without ever calling `os.path.exists` on the resolved target – this avoids leaking host filesystem layout via CI logs (CWE-22 information-disclosure flavor).
4. **Tokens DTCG invariants** – when `tokens` is set, the tokens file parses, the tree is walked recursively (groups recurse into non-`$`-prefixed children; tokens with `$value` walk the value, including composite `$value`s such as gradients, shadows, typography composites, and arrays), and every `{alias.path}` reference inside any string value is resolved to a token (not a group; DTCG forbids aliasing groups).
5. **`tokensContract` enforcement** – each manifest may declare a `tokensContract` mapping dotted token paths (relative to the brand-id wrapping group in `tokens.tokens.json`) to expected DTCG `$type`s. When omitted, the **default contract** applies (see `brands/README.md`): `color.brand.{primary, accent, surface-dark, text-light}` of `$type: color`, plus `typography.fontFamily.{primary, display, mono}` of `$type: fontFamily`. Gate 12 verifies every contract entry resolves to a token whose effective `$type` (resolved via DTCG inheritance – closest ancestor wins) matches the declared type. This is the LSP-substitutability contract – every brand exposes the same semantic role names so consumers can swap brands at runtime without breaking.
6. **Brand `CLAUDE.md` presence** – every non-exempt brand must ship a `CLAUDE.md` at its folder root. This is the prose Claude entry point for brand-styled artwork: asset-library catalog pointers, hard caveats the manifest cannot express, when-to-use guidance per deliverable type. The bootstrap (`skills/using-erfana/SKILL.md`) reads it generically, so adding a new brand requires zero sub-skill changes. `example-acme` is exempt as a documented placeholder (`BRAND_STRUCTURE_EXEMPT` set in the gate); WIP brands prefixed with `_` are skipped entirely.
7. **Per-library `INDEX.md` presence** – for each declared `imagery.{photoLibrary, backgroundLibrary, shapeLibrary, templateLibrary, logoLibrary}` directory, plus the `logo/` folder when derived from `logos[].src`, an `INDEX.md` must exist at the library root. Subfolders at any depth that contain their own `INDEX.md` (e.g. `templates/slides/INDEX.md` while `templates/INDEX.md` is a thin pointer) are also validated – this is how the chained-INDEX pattern for templates is supported without forcing a file move. Hidden directories (`.git`, `.DS_Store-as-dir`) are pruned during the walk; symlinks are not followed (`os.walk` default).
8. **Bidirectional `INDEX.md` cross-checks** – every backtick-quoted file path in any `INDEX.md` must resolve to a real file in the library tree (cite → file). Conversely, every non-`INDEX.md` file in the library tree must be mentioned by at least one `INDEX.md` in the chain (file → cited). The second direction also covers any `RULES.md` co-located with INDEX.md – a RULES.md that exists but is uncited fails this check, which is what guarantees per-library deep-prose docs stay discoverable from their catalogue. The first direction catches typos and stale references after asset removal; the second catches silent drift when a designer adds an asset without updating the catalog. Cite resolution is routed through the same `safe_resolve` used for manifest paths, so a cite that uses `..` to escape the repository emits a sanitized error and never probes the host filesystem (defense-in-depth against using INDEX.md as an existence-disclosure oracle). Glob patterns (`*`, `?`, `[`) and whitespace-bearing strings are filtered out of the cite-detection pass to avoid false positives on prose like `*.svg`. Filename comparisons normalise both sides to Unicode NFC, so Polish diacritics like `ń` (U+0144 vs `n` + U+0303) on macOS APFS do not produce false negatives.
9. **`CLAUDE.md`-references-`INDEX.md` cross-reference** – the brand's `CLAUDE.md` must cite at least one `INDEX.md` per declared library (via substring match on the relative path). Catches the case where a designer adds a new asset library + INDEX.md but forgets to wire it into the brand's prose entry point.
10. **`CLAUDE.md`-references-`RULES.md` cross-reference (v0.4.0+)** – every `RULES.md` discovered inside any library directory must also be cited from the brand-root `CLAUDE.md` (substring match on the path relative to the brand folder). Mirrors check #9 for the deep-prose layer. The INDEX.md → RULES.md edge is enforced incidentally by Direction B of check #8; this completes the symmetry so RULES.md is discoverable from BOTH the per-library catalogue AND the brand-root prose entry point. Closes the orphan risk where a sub-skill that jumps straight from `brand.json` to a specific `INDEX.md` would otherwise skip the rules layer.

## Implementation

The gate is a standalone script so it can be invoked independently of the full runner:

```bash
bash scripts/gate-12-brand-manifests.sh
```

## Pass criteria

`PASS: <N> brand manifest(s) valid; ACTIVE_BRAND="<id>" ok` with `N >= 1`.

## Adding a brand

1. Create `brands/<id>/` – copy `example-acme/` for a minimal manifest, or `erfana/` for the default logo-only worked example (manifest, tokens, `CLAUDE.md`, and a single `logo/` library with its `INDEX.md`). A brand that needs the fuller asset-library scaffold (v1.2 directories `backgrounds/`, `shapes/`, `templates/`, the v1.3 `logoLibrary` pointer, and a populated per-library `INDEX.md` set) builds those out per the schema; the default `erfana` brand intentionally ships none of them.
2. Author `brands/<id>/CLAUDE.md` – the brand's prose Claude entry point. Required sections (see `brands/README.md` for the canonical list): brand identity recap, asset-library catalog pointing at every `INDEX.md` the brand ships, when-to-consult-what per deliverable type, hard caveats the manifest cannot express, manifest source-of-truth pointer.
3. For every declared `imagery.*Library` directory and the `logo/` folder, ship an `INDEX.md` at the library root. Format: top heading, prose intro, sectioned tables with `File | What's on it | When to use` columns, footer with `Version` / `Created` / `Applies to` lines. Audit each new INDEX.md with the **two-reviewer review protocol** documented in `brands/README.md` before merging.
4. Edit `scripts/gate-12-brand-manifests.sh` `PRODUCTION_BRANDS` to add the new id (only when the brand is ready to be active-eligible – placeholder / staged brands stay off the allowlist). Documented placeholders may be added to `BRAND_STRUCTURE_EXEMPT` if they should not enforce the CLAUDE.md / INDEX.md contract (currently `example-acme` only).
5. Optionally edit `ACTIVE_BRAND` to point at the new brand for a one-off render.
6. Run `bash scripts/run-all-gates.sh`. Gate 12 validates the new brand against the schema, the path-traversal guard, the tokensContract, AND the CLAUDE.md / INDEX.md presence + cross-checks introduced this session.

No skill code needs to change. Discovery is via filesystem glob; the only edit outside the new folder is the allowlist + (optionally) `ACTIVE_BRAND`. Brand inventory today: `erfana` (active, on `PRODUCTION_BRANDS`; neutral logo-only default bundle – `brand.json`, `tokens.tokens.json`, `CLAUDE.md`, and a single `logo/` library with its `INDEX.md`. No photos, shapes, backgrounds, templates, brandbook, or `RULES.md`), `example-acme` (placeholder; documented exception, off allowlist, exempt from CLAUDE.md / INDEX.md checks).
